API keys

These keys authenticate every request to /api/* — the same internal API the MCP server (used by Claude) and any other integration go through. Each request must present a key as Authorization: Bearer <key>; keys are stored only as a one-way hash and shown in full exactly once, at creation, so make sure to copy it down then.

Every key belongs to a real, active user, and every action taken with it — through the MCP server or otherwise — is recorded in the activity log under that person, not a shared account. Give each person (or each client, e.g. a script) their own key rather than sharing one, so the log reflects who actually did what. Revoking a key takes effect immediately, with no restart needed, and does not affect any other key. Deleting a key removes it permanently (its past activity-log entries are unaffected); a person can't have two active keys with the same label, so revoke or rename before reusing one.
